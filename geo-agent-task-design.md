# Geo Agent Task Design

## Overview

基于 Meta ARE (Agent Research Environments) 框架，设计一套面向真实地理世界的 Agent Benchmark 评测任务。Agent 通过 Google Maps API 与真实世界交互，完成从简单查询到复杂多步规划的地理推理任务。

**目标**: Benchmark 评测，量化 Agent 的地理推理能力
**数据接口**: Google Maps 全套 API (Directions, Places, Geocoding, Distance Matrix, Elevation, Traffic)
**模态**: 文本 + 图片（多模态输入）
**任务数量**: 19 个，分 5 个层级

---

## Capability Tags

| Tag | Description |
|-----|-------------|
| `SingleTool` | 单个 API 调用即可完成 |
| `MultiStep` | 需要多个 API 组合调用 |
| `Constraint` | 需要满足特定约束条件 |
| `DynamicEvent` | 需要应对运行中注入的动态事件 |
| `Optimization` | 需要路线或方案优化 |
| `Comparison` | 需要多方案比较和推荐 |
| `TimePlanning` | 需要时间维度的规划 |
| `Vision` | 需要图片视觉理解 |

## Layer 1: Single Tool Call (基础能力)

### Task 1: 地址解析

- **Tags**: `SingleTool`
- **API**: Geocoding
- **Description**: 给定一个自然语言地址，返回经纬度坐标。

**示例输入**:

```
用户: 北京市海淀区中关村大街1号的经纬度是多少？
```

**示例输出**:

```
Agent: 北京市海淀区中关村大街1号的经纬度为：
- 纬度: 39.9836
- 经度: 116.3161
该位置位于中关村核心区域，靠近中关村地铁站。
```

**验证逻辑**: 返回的经纬度与标准答案误差 < 0.01

---

### Task 2: 两点距离查询

- **Tags**: `SingleTool`
- **API**: Distance Matrix
- **Description**: 查询两点之间的距离和预估通勤时间。

**示例输入**:

```
用户: 从上海虹桥机场到浦东机场有多远？开车要多久？
```

**示例输出**:

```
Agent: 从上海虹桥机场到浦东机场的驾车信息：
- 距离: 约 56.3 公里
- 预计用时: 约 50 分钟（当前路况）
注意：实际用时会受交通状况影响，高峰期可能需要 1.5-2 小时。
```

**验证逻辑**: 返回了距离（km）和时长（min），且数值在合理范围内（距离 40-70km，时间 30-120min）

### Task 3: 地点搜索

- **Tags**: `SingleTool`
- **API**: Places (Nearby Search)
- **Description**: 在指定位置和半径内搜索特定类型的地点。

**示例输入**:

```
用户: 故宫附近1公里内有哪些咖啡厅？
```

**示例输出**:

```
Agent: 故宫（北京）附近1公里内找到以下咖啡厅：
1. 故宫角楼咖啡 - 评分 4.5 - 距离约 200m
2. Starbucks 东华门店 - 评分 4.2 - 距离约 650m
3. Manner Coffee 南池子店 - 评分 4.3 - 距离约 800m
...
```

**验证逻辑**: 返回结果全部在 1km 半径内，且类型为 cafe/coffee

### Task V1: 景点识别与定位

- **Tags**: `SingleTool`, `Vision`
- **API**: Geocoding
- **Description**: 根据一张景点图片，识别景点名称并返回地址和坐标。

**示例输入**:

```
用户: [附件: eiffel_tower.jpg] 这是哪里？给我它的地址和经纬度。
```

**示例输出**:

```
Agent: 这是法国巴黎的埃菲尔铁塔（Eiffel Tower）。
- 地址: Champ de Mars, 5 Av. Anatole France, 75007 Paris, France
- 纬度: 48.8584
- 经度: 2.2945
- 开放时间: 09:30-23:45（具体以官网为准）
```

**验证逻辑**: 正确识别景点名称，返回的经纬度与标准答案误差 < 0.01

### Task V2: 街景方向判断

- **Tags**: `Vision`, `MultiStep`
- **API**: Places, Directions
- **Description**: 根据一张街景照片，推断用户所在位置，并给出到最近目标的路线。

**示例输入**:

```
用户: [附件: street_view.jpg —— 照片中可见"王府井大街"路牌和东方新天地商场]
     我现在站在这里，最近的地铁站怎么走？
```

**示例输出**:

```
Agent: 根据照片中的"王府井大街"路牌和东方新天地商场，您应该在王府井大街南端。
最近的地铁站是 王府井站（1号线），步行约 3 分钟：
1. 沿王府井大街向北步行 200 米
2. 在路东侧可看到地铁入口（A口）
```

**验证逻辑**: 正确识别位置（王府井区域），推荐的地铁站合理，步行路线方向正确

## Layer 2: Multi-Step Reasoning (组合能力)

### Task 4: 基础路线规划

- **Tags**: `MultiStep`, `Constraint`
- **API**: Directions
- **Description**: 规划满足特定约束的路线（如避开高速、避开收费站）。

**示例输入**:

```
用户: 帮我规划从北京大学到天安门的驾车路线，避开高速。
```

**示例输出**:

```
Agent: 从北京大学到天安门的驾车路线（已避开高速）：
- 总距离: 14.2 公里
- 预计用时: 约 35 分钟

路线：
1. 从北京大学东门出发，沿中关村北大街向南
2. 右转进入海淀南路
3. 沿西二环辅路向南
4. 经西长安街到达天安门

注意：该路线全程走城市道路，不经过高速公路。
```

**验证逻辑**: 返回的路线中不包含高速公路路段，起终点正确

### Task 5: 多模式出行对比

- **Tags**: `MultiStep`, `Comparison`
- **API**: Directions (driving/transit/bicycling)
- **Description**: 比较多种出行方式，给出推荐。

**示例输入**:

```
用户: 从西二旗地铁站到国贸，开车、地铁、骑车分别要多久？推荐哪种？
```

**示例输出**:

```
Agent: 从西二旗到国贸的三种出行方式对比：

| 方式 | 距离 | 用时 | 费用估算 |
|------|------|------|----------|
| 驾车 | 22km | 45-80min | 停车费约30-50元 |
| 地铁 | - | 约55min | 7元 |
| 骑车 | 24km | 约90min | 免费 |

推荐：地铁。
理由：虽然驾车最快（不堵车时），但早晚高峰西二旗到国贸方向严重拥堵，
实际用时可能超过80分钟。地铁用时稳定，且无需考虑停车问题。
```

**验证逻辑**: 三种方式都查询了，推荐结果包含合理依据（对比分析）

### Task 6: 途经点路线优化

- **Tags**: `MultiStep`, `Optimization`
- **API**: Directions (waypoints + optimize)
- **Description**: 给定多个途经点，规划最优访问顺序。

**示例输入**:

```
用户: 我要从酒店（王府井）出发，先去故宫，再去颐和园，最后去首都机场，帮我规划最优驾车路线。
```

**示例输出**:

```
Agent: 最优驾车路线（已优化途经顺序）：

王府井酒店 → 故宫（5min, 2km）
  → 颐和园（40min, 18km）
  → 首都机场 T3（45min, 32km）

总距离: 约 52 公里
总用时: 约 1 小时 30 分钟

优化说明：按地理位置从市中心向西北再向东北，避免来回折返。
原始顺序和优化顺序一致，无需调整。

提示：故宫周一闭馆，请注意安排日期。
```

**验证逻辑**: 所有途经点都包含在路线中，总路程优于未优化的顺序

### Task V3: 相似景点推荐

- **Tags**: `Vision`, `MultiStep`
- **API**: Places
- **Description**: 根据图片识别景点特征，在用户指定区域推荐相似地点。

**示例输入**:

```
用户: [附件: karst_landscape.jpg —— 桂林漓江山水照片]
     我不知道这是哪个景点，但我很喜欢这种风格，我现在在成都，想在周边找个类似的地方旅行。
```

**示例输出**:

```
Agent: 这张图片是广西桂林漓江的喀斯特地貌山水风景。

这种地貌的特征是：石灰岩溶蚀形成的峰林、溶洞和碧绿水域。
在成都周边，推荐以下类似风景的目的地：

1. 蜀南竹海（宜宾）- 距成都约 280km，车程 3.5h
   - 特色：竹林覆盖的喀斯特丘陵，有溶洞和溪流

2. 兴文石海（宜宾）- 距成都约 350km，车程 4h
   - 特色：典型喀斯特地貌，石林、溶洞、天坑

3. 九寨沟（阿坝）- 距成都约 440km，车程 7h
   - 特色：虽非喀斯特，但有类似的碧绿水域和山水意境

推荐首选蜀南竹海，距离较近且最接近图片中的山水风格。
```

**验证逻辑**: 正确识别图片中的地貌特征（喀斯特/山水），推荐地点确实在成都周边（< 500km），且具有相似的自然特征

## Layer 3: Constraint Reasoning (条件约束)

### Task 7: 时间约束规划

- **Tags**: `Constraint`, `TimePlanning`
- **API**: Places, Distance Matrix, Directions
- **Description**: 在时间约束下规划"吃饭 + 赴约"的行程。

**示例输入**:

```
用户: 我下午2点在三里屯有个会议，现在（11:30）在清华大学。
     我想先在附近吃个午饭再过去，推荐一个餐厅并规划路线，确保我能准时到。
```

**示例输出**:

```
Agent: 时间分析：
- 可用时间: 11:30 - 14:00 = 2.5 小时
- 清华到三里屯驾车: 约 35 分钟
- 需预留出发时间: 13:20 前出发
- 可用餐时间: 11:30 - 13:20 = 约 1 小时 50 分钟

推荐方案：
1. 步行到清华南门附近的"五道口枣糕王"（10min）
   - 用餐约 40 分钟
2. 12:20 用餐结束
3. 12:30 打车出发前往三里屯
4. 预计 13:05-13:20 到达（留有余量）

备选：清华校内食堂（更快，但非周末可能不对外开放）

路线: 清华南门 → 成府路 → 北四环 → 东三环 → 三里屯
```

**验证逻辑**: 选的餐厅到清华步行/驾车时间合理，餐厅到三里屯的通勤时间 + 用餐时间 < 2.5 小时

```
Agent: 成都→拉萨 318国道骑行海拔剖面：

起点: 成都 - 海拔约 500m
终点: 拉萨 - 海拔约 3650m
总距离: 约 2,140 公里

关键海拔节点：
1. 成都 (500m) → 雅安 (600m) —— 平缓
2. 雅安 → 康定 (2,560m) —— 翻越二郎山
3. 康定 → 理塘 (4,014m) —— 翻越折多山(4,298m)、高尔寺山
4. 理塘 → 巴塘 (2,580m) —— 短暂下降
5. 芒康 → 左贡 → 邦达 (4,334m) —— 翻越东达山

★ 全程最高点: 米拉山口，海拔约 5,013 米

建议：全程需要约 25-30 天，注意高原反应，建议在理塘和邦达各休整1-2天。
```

**验证逻辑**: 返回了海拔剖面数据，最高点识别正确（东达山或米拉山口，约 5000m+）

### Task V5: 地图截图路线纠错

- **Tags**: `Vision`, `Constraint`, `Optimization`
- **API**: Geocoding, Directions
- **Description**: 根据手绘或截图的路线图，提取地点信息，验证并优化路线。

**示例输入**:

```
用户: [附件: hand_drawn_route.jpg —— 手绘路线图，标注了：
      起点"我家（回龙观）" → "西单" → "国贸" → "天坛" → 终点"回龙观"]
     朋友画了这个自驾一日游路线给我，但我觉得顺序不太对，帮我验证一下并优化。
```

**示例输出**:

```
Agent: 我从图中提取了以下路线：
回龙观 → 西单 → 国贸 → 天坛 → 回龙观

问题分析：
当前顺序总行程约 78km，存在明显绕路：
- 西单在西边，国贸在东边，天坛在东南，来回折返

优化后路线：
回龙观 → 天坛 → 国贸 → 西单 → 回龙观

| 路线 | 总距离 | 总时间 |
|------|--------|--------|
| 原始顺序 | 78km | 约 2h40min |
| 优化顺序 | 61km | 约 2h05min |

节省: 17km / 35 分钟

优化逻辑：从北向南先到天坛，再向东到国贸，最后向西到西单，
形成顺时针环线，避免东西折返。
```

**验证逻辑**: 正确提取了图中的 4 个地点，优化后的路线总距离 < 原始路线

### Task V6: 停车找车场景

- **Tags**: `Vision`, `DynamicEvent`

```
[t=30s] (动态事件注入)

     我逛到了这里，怎么走回停车场？
```

**示例输出**:

```
Agent: 已定位：
- 停车位置: 朝阳大悦城停车场 B2 层（朝阳北路101号）
- 当前位置: 蓝色港湾（朝阳公园路6号）

步行路线：
1. 从蓝色港湾东门出发
2. 沿朝阳公园路向西步行 500m
3. 右转进入朝阳北路
4. 步行约 800m 到达朝阳大悦城北门
5. 从北门进入商场，乘电梯至 B2 停车场

总距离: 约 1.5 公里
预计步行时间: 约 18 分钟
```

**验证逻辑**: 两张图片的位置都正确识别，返回了合理的步行路线

## Layer 4: Dynamic Event Response (动态事件)

### Task 9: 实时交通应对

- **Tags**: `DynamicEvent`, `Optimization`
- **API**: Directions, Distance Matrix
- **Description**: 导航途中遭遇交通事件，Agent 需主动重新规划路线。

**示例输入**:

```
[t=0s]
用户: 帮我规划从望京到国贸的驾车路线。

[t=10s] (环境事件注入)
系统通知: 实时路况更新 —— 东三环望京桥至国贸桥路段发生交通事故，
         严重拥堵，预计延误 40 分钟。当前路线受影响。
```

**示例输出**:

```
# 第一次响应 (t=0s)
Agent: 从望京到国贸的驾车路线：
- 路线: 望京 → 京密路 → 东三环 → 国贸
- 距离: 12km | 用时: 约 25 分钟

# 第二次响应 (t=10s, 收到交通事件后)
Agent: ⚠️ 收到路况更新：东三环望京桥至国贸桥路段严重拥堵。
已为您重新规划路线：

新路线: 望京 → 机场高速 → 东四环 → 四惠桥 → 国贸
- 距离: 16km | 用时: 约 30 分钟
- 比原路线多 4km，但预计节省约 35 分钟

建议立即切换至新路线。
```

**验证逻辑**: Agent 在交通事件后主动重新规划，新路线避开了拥堵路段，且预计用时 < 原路线受堵后的用时

```
- 朋友B: 通州（东部）

地理中心区域: 朝阳区 CBD - 建国门一带

推荐会合餐厅（川菜，评分 > 4.0）：

1. 眉州东坡（建国门店）⭐ 推荐
   - 评分: 4.5 | 人均: ¥95
   - 到你: 5km, 15min
```
